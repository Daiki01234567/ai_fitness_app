#### 4.2.2 カメラ映像の取り扱い（重要）

**カメラ映像は端末上でのみ処理され、サーバーには送信されません。**

当社は、EDPBガイドライン「データ保護バイデザインおよびバイデフォルト」に基づき、プライバシー保護のため、以下の技術的措置を講じています：

| 項目 | 技術的措置 | プライバシー保護 |
|-----|-----------|----------------|
| **カメラ映像** | 端末上でのみ処理（オンデバイス処理） | サーバーに送信しない |
| **骨格座標データ** | 数値データのみを抽出してサーバーに保存 | 個人識別不可 |
| **個人識別** | 骨格座標データから個人を識別することはできません | 匿名データ |
| **映像の保存** | カメラ映像は端末内で処理後、即座に破棄 | メモリに残らない |
| **サーバー保存** | 骨格座標データ（数値）のみ | 映像データは保存しない |

**技術的な詳細**:

1. **MediaPipe Poseの仕組み**:
   - カメラ映像から骨格座標（関節の位置情報）のみを抽出します
   - 骨格座標データは、以下の数値データで構成されます：
     - X座標（0.0〜1.0の正規化された値）
     - Y座標（0.0〜1.0の正規化された値）
     - Z座標（深度情報、相対値）
     - 信頼度スコア（0.0〜1.0）

2. **個人識別不可の根拠**:
   - 骨格座標データは、関節の位置関係を示す数値データであり、顔、体型、肌の色等の個人を特定できる情報は含まれません
   - カメラ映像そのものは保存されないため、個人を識別することはできません

3. **データの流れ**:
   ```
   カメラ映像（端末）
         ↓ オンデバイス処理
   骨格座標データ抽出（端末）
         ↓ 数値データのみ送信
   サーバー保存（Firebase/BigQuery）
         ↓
   カメラ映像は即座に破棄（メモリから削除）
   ```

4. **セキュリティ対策**:
   - カメラ映像は端末のメモリ上でのみ一時的に保持されます
   - MediaPipe Pose処理後、カメラ映像は即座にメモリから削除されます
   - 骨格座標データの送信は、HTTPS（TLS 1.3以上）で暗号化されます

#### 4.2.3 骨格座標データの具体例

**データ形式**（JSON）:
```json
{
  "timestamp": "2025-05-01T10:30:00Z",
  "sessionId": "abc123",
  "landmarks": [
    {
      "name": "LEFT_SHOULDER",
      "x": 0.45,
      "y": 0.32,
      "z": -0.15,
      "visibility": 0.98
    },
    {
      "name": "RIGHT_SHOULDER",
      "x": 0.55,
      "y": 0.33,
      "z": -0.14,
      "visibility": 0.97
    }
    // ... 他の関節（全33点）
  ]
}
```

- 集計値のみを使用（平均値、中央値、分散等）

**例**:

- ✅ 匿名化データ: 「2025年5月のスクワット平均回数は12回だった」

### 8.4 アクセス制御

#### 8.4.1 ユーザーデータのアクセス制御

**Firebase Security Rules**による保護：

```javascript
// Firestoreセキュリティルール
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // ユーザーは自分のデータのみアクセス可能
    match /users/{userId} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;
    }

    // セッションデータは本人のみアクセス可能
    match /sessions/{sessionId} {
      allow read: if request.auth != null
        && resource.data.userId == request.auth.uid;
      allow create: if request.auth != null
        && request.resource.data.userId == request.auth.uid;
    }

    // 種目データは認証済みユーザーのみ読み取り可能
    match /exercises/{exerciseId} {
      allow read: if request.auth != null;
      allow write: if false; // 管理者のみCloud Functionsから書き込み
    }
  }
}
```

**アクセス制御の原則**:
1. **認証必須**: すべてのデータアクセスに認証が必要
2. **本人のみアクセス**: ユーザーは自分のデータのみアクセス可能
3. **書き込み制限**: 管理用データは管理者のみ書き込み可能

### 9.3 削除権（Right to Erasure / Right to be Forgotten / GDPR第17条）

#### 9.3.1 権利の内容（忘れられる権利）

4. **個人情報が違法に処理された場合**
5. **EU法または加盟国法上の法的義務の遵守のために削除が必要な場合**
6. **子供に関する個人情報が情報社会サービスの提供に関連して収集された場合**（GDPR第8条1項）

## 10. Cookieの使用

### 10.1 Cookieとは

Cookieとは、Webサイトやアプリが利用者のデバイスに保存する小さなテキストファイルです。Cookieにより、ユーザーの設定や利用状況を記憶することができます。

### 10.2 本サービスで使用するCookie

本サービスでは、以下のCookieを使用します：

#### 10.2.1 必須Cookie

| Cookie名 | 提供者 | 目的 | 保存期間 | オプトアウト |
|---------|--------|------|---------|------------|
| **認証Cookie** | Firebase Authentication | ログイン状態の維持 | 30日間 | 不可（サービス提供に必須） |
| **セッションCookie** | Firebase | セッション管理 | セッション終了まで | 不可（サービス提供に必須） |

**法的根拠**: 契約の履行（GDPR第6条1項(b)）

#### 10.2.2 分析Cookie

|---------|--------|------|---------|------------|

| **Crashlytics** | Google | クラッシュ分析 | 90日間 | 可能（設定から変更） |

**法的根拠**: 同意（GDPR第6条1項(a)）

#### 10.2.3 Cookieバナー

EEA居住者に対しては、初回アクセス時にCookieバナーを表示し、分析Cookieの使用について同意を取得します（eプライバシー指令）。

### 10.3 第三者Cookie

本サービスでは、以下の第三者Cookieを使用する場合があります：

| Cookie名 | 提供者 | 目的 | オプトアウト |
|---------|--------|------|------------|

### 10.4 Cookieの管理

#### 10.4.1 アプリ内での設定

### 12.2 EUから日本へのデータ転送

#### 12.2.1 十分性認定

**十分性認定決定**: Commission Implementing Decision (EU) 2019/419 of 23 January 2019

**意味**: EUから日本への個人情報の転送は、追加的な保護措置なしに可能です。

### 12.3 EUから米国へのデータ転送

#### 12.3.1 法的根拠

EEAの利用者の個人情報を米国に転送する場合、以下の適切な保護措置を講じています：

| 保護措置 | 詳細 |
|---------|------|
| **EU-US Data Privacy Framework** | Googleは認証済み（DPF List: https://www.dataprivacyframework.gov/） |
| **Standard Contractual Clauses（SCC）** | Google Cloud Data Processing Addendum（DPA）に含まれる |
| **追加的な保護措置** | 暗号化（TLS 1.3、AES-256）、アクセス制御（IAM）、監査ログ |

#### 12.3.2 Schrems II判決への対応

当社は、Schrems II判決（CJEU C-311/18）を踏まえ、以下の対応を行っています：

1. **データ転送影響評価（TIA: Transfer Impact Assessment）の実施**
   - 米国での公的機関によるアクセスのリスク評価
   - 追加的な技術的・組織的措置の必要性評価
   - 結論: 追加措置により十分な保護が可能

2. **追加的な技術的措置**
   - **エンドツーエンド暗号化**: TLS 1.3による通信暗号化
   - **保存時の暗号化**: AES-256による暗号化
   - **仮名化（pseudonymization）**: ユーザーIDの仮名化

3. **追加的な組織的措置**
   - **アクセス制御の強化**: IAMによる最小権限の原則
   - **監査ログ**: すべてのアクセスをログ記録
   - **定期監査**: 四半期ごとのセキュリティ監査

4. **米国での公的機関によるアクセスのリスク**
   - FISA 702、Executive Order 12333等の米国法によるアクセスのリスクは存在
   - しかし、当社のデータは暗号化されており、技術的にアクセスは困難
   - Googleは政府要請に対する透明性レポートを公開

#### 12.3.3 Standard Contractual Clauses（SCC）

当社は、欧州委員会承認のSCC（2021年版）を使用しています：

- **SCC Module**: Module Two（Controller to Processor）
- **締結先**: Google Cloud（Data Processing Addendum）
- **追加条項**: Schrems II判決に対応した追加条項を含む

### 12.4 その他の国への転送

その他の国への個人情報の転送を行う場合、GDPR第44条〜第50条に基づき、適切な保護措置を講じます：

1. **十分性認定国への転送**: 追加措置不要
2. **その他の国への転送**: SCCの締結、または利用者の明示的な同意

## 13. データ侵害通知

### 13.1 データ侵害とは

データ侵害とは、個人情報の紛失、破壊、改ざん、不正アクセス、不正開示等を指します（GDPR第4条12項）。

### 13.2 監督機関への通知

#### 13.2.1 通知義務

データ侵害が発生し、個人の権利および自由に対してリスクが生じる場合、当社は**72時間以内**に監督機関に通知します（GDPR第33条）。

#### 13.2.2 通知先

| 地域 | 通知先 |
|-----|--------|
| **日本** | 個人情報保護委員会（https://www.ppc.go.jp/） |
| **EEA** | 主たる拠点のあるEEA加盟国のデータ保護機関 |

#### 13.2.3 通知内容

通知には、以下の情報が含まれます（GDPR第33条3項）：

1. **データ侵害の性質**（影響を受けたデータ主体のカテゴリー、数等）
2. **データ保護責任者（DPO）の連絡先**
3. **データ侵害の影響の説明**
4. **講じた対策または講じる予定の対策**

### 13.3 利用者への通知

#### 13.3.1 通知義務

データ侵害が個人の権利および自由に対して**高いリスク**を生じさせる場合、当社は遅滞なく利用者に通知します（GDPR第34条）。

**高いリスクの例**:
- 個人を識別できる情報（氏名、メールアドレス等）の漏洩
- 財務情報の漏洩
- 健康情報の漏洩
- パスワードの漏洩（暗号化されていない場合）

#### 13.3.2 通知方法

| 方法 | 詳細 |
|-----|------|
| **アプリ内通知** | プッシュ通知 |
| **メール** | 登録メールアドレス宛 |
| **公式Webサイト** | 重大な場合、多数の利用者が影響を受ける場合 |

#### 13.3.3 通知内容

通知には、以下の情報が含まれます（GDPR第34条2項）：

1. **データ侵害の性質**
2. **データ保護責任者（DPO）の連絡先**
3. **データ侵害の影響の説明**
4. **講じた対策**
5. **利用者が取るべき対策**（例：パスワード変更）

#### 13.3.4 通知の例外

以下の場合、利用者への通知は不要です（GDPR第34条3項）：

1. **適切な技術的・組織的保護措置を講じている場合**（例：暗号化）
2. **事後的に高いリスクを排除した場合**
3. **過度の負担を要する場合**（公的な通知で代替）

### 13.4 インシデント対応プロセス

**72時間対応プロセス**:

| 時間 | 対応内容 |
|-----|---------|
| **0-1時間** | 検知、初動対応チーム招集、被害の封じ込め |
| **1-24時間** | 影響範囲の特定、深刻度の評価 |
| **24-72時間** | 監督機関への通知準備、根本原因の分析 |
| **72時間以内** | 監督機関への通知（GDPR第33条） |
| **遅滞なく** | 利用者への通知（高リスクの場合、GDPR第34条） |
| **1週間以内** | 根本原因の特定 |
| **2週間以内** | 再発防止策の実施 |
