## 1. はじめに

### Claude Codeとは何か

**Claude Code**は、Anthropic社が開発したAIコーディングアシスタントです。ターミナル（コマンドを入力する黒い画面）で動作し、自然な日本語や英語で指示を出すだけで、以下のようなことができます。

- コードを書いてもらう（新しい機能の追加など）
- 既存のコードを説明してもらう
- バグ（プログラムの間違い）を見つけて修正してもらう
- ファイルを作成・編集してもらう
- Gitの操作（コードの履歴管理）を手伝ってもらう

**例えるなら**: Claude Codeは「とても賢いプログラミングの先生」のようなものです。あなたが「ログイン機能を作って」と言えば、どんなコードを書けばいいか教えてくれますし、実際にコードを書いてくれます。

### このガイドの目的

このガイドでは、Claude Codeを使い始めるための環境構築から、効果的な使い方のコツまでを、わかりやすく説明します。プログラミング初心者でも安心して読み進められるように、専門用語には説明を付けています。

## 2. 環境構築

### 前提条件

Claude Codeを使うためには、以下のソフトウェアがパソコンにインストールされている必要があります。

#### Node.jsとnpm

**Node.js**（ノード・ジェイエス）は、JavaScriptというプログラミング言語をパソコン上で動かすための実行環境です。

**npm**（エヌ・ピー・エム）は、Node.jsのパッケージ（便利なプログラムの部品）を管理するツールです。Node.jsをインストールすると、npmも一緒にインストールされます。

**インストール確認方法**:

ターミナル（Windowsの場合はコマンドプロンプトまたはPowerShell）を開いて、以下のコマンドを入力してください。

```bash
node --version
# 例: v18.0.0 以上であればOK

npm --version
# 例: 8.0.0 以上であればOK
```

**インストールがまだの場合**:

[Node.js公式サイト](https://nodejs.org/)から「LTS」版をダウンロードしてインストールしてください。詳しい手順は[開発環境セットアップガイド](./DEVELOPMENT_SETUP_GUIDE.md)を参照してください。

---

### Claude Codeのインストール

Node.jsがインストールされたら、以下のコマンドでClaude Codeをインストールします。

#### Windows（PowerShell推奨）

```powershell
irm https://claude.ai/install.ps1 | iex
```

#### macOS/Linux

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

#### npmを使う方法（全OS共通）

```bash
npm install -g @anthropic-ai/claude-code
```

**補足**: `-g`は「グローバルインストール」の意味で、パソコン全体でコマンドが使えるようになります。

**インストール確認**:

```bash
claude -v
# 例: 2.0.61（バージョン番号が表示されればOK）
```

### 認証設定（APIキー）

Claude Codeを使うには、**APIキー**が必要です。

**APIキー**とは: サービスを利用するための「会員証」のようなものです。これがないとClaude Codeは動きません。

**取得方法**:

1. [Anthropicコンソール](https://console.anthropic.com/)にアクセス
2. アカウントを作成またはログイン
3. 「API Keys」メニューからキーを作成
4. 作成されたキーをコピー

**設定方法**:

初回起動時にAPIキーの入力を求められます。または、環境変数に設定することもできます。

```bash
# Windowsの場合（PowerShell）
$env:ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxxx"

# macOS/Linuxの場合
export ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxxx"
```

> **注意**: APIキーは他人に見せないでください。パスワードと同じくらい大切なものです。

### MCPサーバーの設定（.mcp.json）

**MCP**（Model Context Protocol）は、Claude Codeに追加の機能を与える仕組みです。詳しくは[5.1 MCP](#51-mcpmodel-context-protocol)で説明します。

MCPサーバーの設定は、プロジェクトフォルダに`.mcp.json`というファイルを作成して行います。

**設定例**（`.mcp.json`）:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/files"]
    }
  }
}
```

### Serena MCPのセットアップ

**Serena**は、コードの意味を理解して賢く編集できる強力なMCPサーバーです。大きなプロジェクトでも、目的のコードを素早く見つけて編集できます。

**インストール手順**:

1. まず`uv`（uvx）をインストールします。

```bash
# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows（PowerShell）
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

2. Serenaを起動できるか確認します。

```bash
uvx --from git+https://github.com/oraios/serena serena start-mcp-server --help
```

3. `.mcp.json`に設定を追加します。

```json
{
  "mcpServers": {
    "serena": {
      "command": "uvx",
      "args": ["--from", "git+https://github.com/oraios/serena", "serena", "start-mcp-server"]
    }
  }
}
```

詳しい設定方法は[Serena公式ドキュメント](https://oraios.github.io/serena/02-usage/030_clients.html)を参照してください。

## 3. 基本コマンド

Claude Codeの対話モード中で使える**スラッシュコマンド**を紹介します。

### スラッシュコマンドとは

**スラッシュコマンド**は、`/`（スラッシュ）で始まる特別なコマンドです。Claude Codeに特定の操作を指示できます。

| コマンド | 説明 |
|---------|------|
| `/init` | プロジェクトの初期設定を行います。`CLAUDE.md`ファイルを作成します。 |
| `/mcp` | MCPサーバーの管理を行います。追加・削除・確認ができます。 |
| `/model` | 使用するAIモデルを選択します（sonnet/opus/haiku）。 |
| `/compact` | 会話履歴を圧縮してコンテキストを節約します。 |
| `/agents` | サブエージェントの一覧・管理を行います。 |
| `/help` | 利用可能なコマンドの一覧を表示します。 |
| `/bug` | バグ（不具合）を報告します。 |

---

### `/init` - プロジェクトの初期設定

**何ができるか**: プロジェクト専用の設定ファイル`CLAUDE.md`を作成します。

**使いどころ**: 新しいプロジェクトでClaude Codeを使い始めるとき。

**使い方**:
```
/init
```

**`CLAUDE.md`とは**: プロジェクトの説明やルールをClaude Codeに教えるためのファイルです。これを作っておくと、Claude Codeがプロジェクトの内容を理解して、より適切な回答をしてくれます。

---

### `/mcp` - MCPサーバーの管理

**何ができるか**: MCPサーバーの追加・削除・状態確認ができます。

**使いどころ**: 新しいMCPサーバーを追加したいとき、または既存のサーバーの状態を確認したいとき。

**使い方**:
```
/mcp
```

表示される指示に従って、MCPサーバーを管理できます。

---

### `/model` - AIモデルの選択

**何ができるか**: Claude Codeが使うAIモデル（頭脳の種類）を切り替えます。

**使いどころ**: タスクの難易度に応じてモデルを変えたいとき。

**使い方**:
```
/model
```

**選べるモデル**:

| モデル名 | 特徴 |
|---------|------|
| `sonnet` | バランス型。日常的なコーディングタスクに最適。（標準） |
| `opus` | 最高性能。複雑な設計や難しい問題に強い。 |
| `haiku` | 高速。簡単なタスクや素早い回答が欲しいときに。 |

---

### `/compact` - コンテキストの圧縮

**何ができるか**: 会話履歴を要約して、メモリ（コンテキスト）の使用量を減らします。

**使いどころ**: 長い会話をしていて、「コンテキストが足りない」というエラーが出たとき。

**使い方**:
```
/compact
```

**コンテキストとは**: AIが覚えている会話の内容です。長く会話すると、この「記憶」がいっぱいになります。`/compact`を使うと、重要な部分だけを残して圧縮してくれます。

---

### `/agents` - サブエージェントの管理

**何ができるか**: サブエージェント（専門家チーム）の一覧を表示・管理します。

**使いどころ**: どんなサブエージェントが使えるか確認したいとき。

**使い方**:
```
/agents
```

**サブエージェントとは**: 特定の分野に特化したAIアシスタントです。詳しくは[5.2 Agents（サブエージェント）](#52-agentsサブエージェント)で説明します。

## 4. 起動オプション

Claude Codeを起動するときに、追加のオプションを指定できます。

### 起動オプション一覧

| オプション | 説明 |
|-----------|------|
| `claude --mcp-config=.mcp.json` | MCPサーバー設定ファイルを指定して起動します。 |
| `claude --continue` または `claude -c` | 前回の会話を続きから再開します。 |
| `claude --resume "<session-id>"` または `claude -r` | 特定のセッションIDを指定して再開します。 |
| `claude --dangerously-skip-permissions` | 権限確認をスキップします（危険！開発時のみ）。 |
| `claude --model <model>` | 使用するモデルを指定します（sonnet/opus/haiku）。 |
| `claude -p "質問"` | 質問に回答して終了します（対話モードに入らない）。 |
| `claude -v` | バージョンを表示します。 |

---

### `--mcp-config` - MCP設定ファイルの指定

**何ができるか**: MCPサーバーの設定が書かれたファイルを読み込んで起動します。

**使い方**:
```bash
claude --mcp-config=.mcp.json
```

**使いどころ**: プロジェクトごとに異なるMCPサーバーを使いたいとき。

---

### `--continue` - 前回の会話を続ける

**何ができるか**: 同じフォルダで最後に行った会話の続きから始められます。

**使い方**:
```bash
claude --continue
# または短縮形
claude -c
```

**使いどころ**: 「昨日の作業の続きをしたい」というとき。

---

### `--resume` - 特定のセッションを再開

**何ができるか**: セッションID（会話の識別番号）を指定して、特定の会話を再開します。

**使い方**:
```bash
claude --resume "abc123-def456"
# または短縮形
claude -r "abc123-def456"
```

**使いどころ**: 複数の会話を並行して進めているとき、特定の会話に戻りたい場合。

---

### `--dangerously-skip-permissions` - 権限確認のスキップ

**何ができるか**: ファイルの編集やコマンドの実行時に、確認なしで実行します。

**使い方**:
```bash
claude --dangerously-skip-permissions
```

> **警告**: このオプションは非常に危険です。Claude Codeがファイルを削除したり、危険なコマンドを実行しても、確認なしで実行されてしまいます。
>
> **使っていいのは**:
> - テスト環境でのみ
> - 大切なファイルがない場所でのみ
> - 自分が何をしているか完全に理解しているときのみ

## 5. 重要な概念の解説

Claude Codeを効果的に使うために、いくつかの重要な概念を理解しましょう。

---

### 5.1 MCP（Model Context Protocol）

#### MCPとは何か

**MCP**（エム・シー・ピー）は、「Model Context Protocol」の略で、Claude Codeに追加の能力を与える仕組みです。

**例えるなら**: MCPは「スマートフォンのアプリ」のようなものです。スマートフォン（Claude Code）に新しいアプリ（MCPサーバー）をインストールすることで、できることが増えます。

#### MCPサーバーの役割

MCPサーバーは、Claude Codeに以下のような追加機能を提供します。

- **ファイルシステムアクセス**: 特定のフォルダのファイルを読み書きする
- **データベース接続**: データベースに直接クエリを実行する
- **外部API連携**: 他のサービス（GitHub、Slackなど）と連携する
- **高度なコード分析**: コードの意味を深く理解する（Serenaなど）

#### .mcp.jsonの設定例

プロジェクトフォルダに`.mcp.json`ファイルを作成して、MCPサーバーを設定します。

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "./src"],
      "description": "srcフォルダ内のファイルにアクセスできるようにする"
    },
    "serena": {
      "command": "uvx",
      "args": ["--from", "git+https://github.com/oraios/serena", "serena", "start-mcp-server"],
      "description": "コードの意味を理解して賢く編集する"
    }
  }
}
```

**設定の読み方**:
- `"filesystem"`: サーバーの名前（自分で決めてOK）
- `"command"`: 実行するコマンド
- `"args"`: コマンドに渡す引数（オプション）
- `"description"`: 説明（なくてもOK）

### 5.2 Agents（サブエージェント）

#### サブエージェントとは

**サブエージェント**は、特定の分野に特化した「専門家」のようなAIです。

**例えるなら**: 病院には内科医、外科医、眼科医など、専門分野の異なる医師がいますよね。サブエージェントも同じで、「テストの専門家」「セキュリティの専門家」「ドキュメントの専門家」など、得意分野が異なります。

#### 利用可能なエージェントの種類

このプロジェクトでは、以下のサブエージェントが定義されています（CLAUDE.mdより）。

| エージェント名 | 役割 | 使う場面 |
|---------------|------|----------|
| `flutter-expert` | Flutter開発専門家 | アプリの画面を作るとき |
| `typescript-pro` | TypeScript開発専門家 | サーバー側のコードを書くとき |
| `qa-expert` | テスト自動化スペシャリスト | テストコードを書くとき |
| `cloud-architect` | クラウドアーキテクト | Firebaseの設定をするとき |
| `git-workflow-manager` | Gitワークフロー専門家 | コードの履歴管理をするとき |
| `security-auditor` | セキュリティ監査専門家 | セキュリティをチェックするとき |
| `documentation-engineer` | 技術ドキュメント専門家 | ドキュメントを作成するとき |
| `error-detective` | エラー分析エキスパート | バグを調査するとき |

#### 「タスクは全てサブエージェントに委譲」の意味

「タスクを委譲する」とは、「仕事を任せる」という意味です。

メインのClaude Codeに「このタスクをやって」と頼むと、Claude Codeは最適なサブエージェント（専門家）を選んで、その仕事を任せます。

**例**:
```
あなた: 「ログイン画面のテストを書いて」
Claude Code: 「これはテストの仕事だな。qa-expertに任せよう」
→ qa-expert（テストの専門家）がテストコードを書く
```

このように、適切な専門家に仕事を任せることで、より質の高い結果が得られます。

### 5.3 コンテキスト

#### コンテキストとは（会話の記憶）

**コンテキスト**は、AIが覚えている「会話の内容」のことです。

**例えるなら**: 友達と話しているとき、「さっき言ってた○○の件だけど...」と言えば、友達は何のことか分かりますよね。これは友達が会話の「コンテキスト（文脈）」を覚えているからです。AIも同じように、会話の流れを覚えています。

#### コンテキストウィンドウの制限

AIが一度に覚えられる情報には限りがあります。これを「コンテキストウィンドウ」と呼びます。

**例えるなら**: 机の上に置ける書類の量には限りがありますよね。AIも同じで、一度に「見える」情報量には限界があります。

**制限に達するとどうなるか**:
- 会話の最初の方の内容を忘れてしまう
- 「コンテキストが足りません」というエラーが出る

#### `/compact`でのコンテキスト管理

`/compact`コマンドを使うと、会話の内容を要約して、コンテキストの使用量を減らせます。

**いつ使うか**:
- 長い会話をしていて、応答が遅くなってきたとき
- 「コンテキストが足りません」というエラーが出たとき
- 新しい話題に移りたいが、重要な内容は覚えておいてほしいとき

```
/compact
```

### 5.4 Serena MCP

#### Serenaとは

**Serena**は、コードの「意味」を理解できる高性能なMCPサーバーです。

普通のテキスト検索では、「login」という文字列を検索することしかできません。しかしSerenaを使うと、「ログイン機能を担当している関数」「認証に関連するクラス」など、コードの意味を理解した検索・編集ができます。

**例えるなら**: 図書館で本を探すとき、「タイトルに"ログイン"が含まれる本」を探すのと、「ログイン機能の作り方が書いてある本」を探すのでは、見つかる本が違いますよね。Serenaは後者のような「意味を理解した検索」ができます。

#### `/serena`コマンドの使い方

Serena MCPを設定した後、Claude Code内で`/serena`コマンドを使って高度な操作ができます。

**基本的な使い方**:
```
/serena
```

#### オプション

Serenaには以下のようなオプションがあります。

| オプション | 説明 |
|-----------|------|
| `-q` | クイックモード。素早く結果を返します。 |
| `-d` | デバッグモード。詳細な情報を表示します。 |
| `-c` | コンテキストを含める。周辺のコードも一緒に表示します。 |
| `-s` | サイレントモード。出力を最小限にします。 |
| `-v` | 詳細モード。より多くの情報を表示します。 |
| `-r` | 再帰的検索。サブディレクトリも含めて検索します。 |
| `-t` | タイプ情報を含める。変数や関数の型情報を表示します。 |

#### パターン（使い方のパターン）

Serenaには、よく使う作業パターンが用意されています。

| パターン | 説明 | 使う場面 |
|---------|------|----------|
| `debug` | デバッグ支援 | バグを見つけて修正するとき |
| `design` | 設計支援 | 新しい機能の設計を考えるとき |
| `implement` | 実装支援 | 新しいコードを書くとき |
| `review` | レビュー支援 | コードをチェックするとき |

**使い方の例**:
```
/serena implement 「ユーザー登録機能」
```

## 6. 効果的な指示の書き方

Claude Codeに良い結果を出してもらうには、良い指示を書くことが大切です。

### 6.1 基本テンプレート

以下のテンプレートを使うと、Claude Codeに効果的な指示を出せます。

```
/serena

# 目的
[何をしたいかを1行で書く]

## やってほしいこと
- [具体的なタスク1]
- [具体的なタスク2]
- [具体的なタスク3]

## 制約
- `CLAUDE.md`の制約を遵守する
- タスクは全てサブエージェントに委譲
- [その他の制約があれば追加]

## 参考情報
- [関連するファイルパスや仕様書があれば記載]
```

**テンプレートの使い方**:

1. `[...]`の部分を自分のタスクに合わせて書き換える
2. 不要な項目は削除してOK
3. 必要に応じて項目を追加してOK

---

### 6.2 実際の指示例

#### 例1: コード実装の例

```
/serena

# 目的
ユーザー登録画面のバリデーション（入力チェック）機能を実装する

## やってほしいこと
- メールアドレスの形式チェック
- パスワードの強度チェック（8文字以上、英数字混合）
- 確認用パスワードの一致チェック
- エラーメッセージの日本語表示
```

## 7. ベストプラクティス

Claude Codeを効果的に使うためのコツを紹介します。

### 基本ルール

1. **「/serena」を必ず使う**
   - Serena MCPを設定している場合、最初に`/serena`を入力すると、コードの理解度が上がります。

2. **「タスクは全てサブエージェントに委譲」を必ず入れる**
   - この一文を入れることで、適切な専門家エージェントにタスクが割り当てられます。

3. **具体的な制約を明記する**
   - 「CLAUDE.mdの制約を遵守」と書くことで、プロジェクトのルールを守ってくれます。
   - 他にも守ってほしいルールがあれば、明記しましょう。

4. **中高生レベルで記載を指定する**
   - ドキュメントやコメントを書いてもらうとき、「中高生レベルでわかりやすく」と指定すると、専門用語を避けた説明を書いてくれます。

5. **参考情報を添える**
   - 関連するファイルパスや仕様書を明記すると、より正確な回答が得られます。

### やってはいけないこと

1. **曖昧な指示を出す**
   - 悪い例: 「いい感じに直して」
   - 良い例: 「ログイン画面のボタンの色を青から緑に変更して」

2. **一度に大量のタスクを頼む**
   - 1回の指示では1つのタスクに絞りましょう。
   - 複数のタスクがある場合は、分けて依頼しましょう。

3. **確認なしで本番環境を操作する**
   - `--dangerously-skip-permissions`は開発環境でのみ使用
   - 本番環境では必ず確認を行う

## 8. トラブルシューティング

### よくある問題と解決方法

#### 問題1: 「APIキーが無効です」というエラー

**原因**: APIキーが正しく設定されていない、または有効期限が切れている。

**解決方法**:
1. [Anthropicコンソール](https://console.anthropic.com/)でAPIキーを確認
2. 新しいキーを作成して再設定
3. 環境変数が正しく設定されているか確認

```bash
# 環境変数の確認
echo $ANTHROPIC_API_KEY
```

---

#### 問題2: 「コンテキストが足りません」というエラー

**原因**: 会話が長くなりすぎて、AIの記憶容量を超えた。

**解決方法**:
1. `/compact`コマンドで会話を圧縮
2. 新しいセッションを開始（`claude`コマンドで再起動）
3. 不要な情報を削除して、必要な情報だけを伝え直す

---

#### 問題3: MCPサーバーが接続できない

**原因**: MCPサーバーの設定が間違っている、または必要なソフトウェアがインストールされていない。

**解決方法**:
1. `.mcp.json`の設定を確認
2. 必要なコマンド（`npx`、`uvx`など）がインストールされているか確認
3. `/mcp`コマンドでサーバーの状態を確認

```bash
# npxが使えるか確認
npx --version

# uvxが使えるか確認
uvx --version
```

---

#### 問題4: 「claude」コマンドが見つからない

**原因**: Claude Codeがインストールされていない、またはPATHに追加されていない。

**解決方法**:
1. インストールを再実行
2. ターミナルを再起動
3. 手動でPATHを設定

```bash
# インストール確認
claude -v

# 見つからない場合は再インストール
npm install -g @anthropic-ai/claude-code
```

---

#### 問題5: 日本語が文字化けする

**原因**: ターミナルの文字コード設定が正しくない。

**解決方法**:
- **Windows**: PowerShellで `chcp 65001` を実行してUTF-8に設定
- **macOS/Linux**: 通常は問題ないが、`export LANG=ja_JP.UTF-8` を設定

---

#### 問題6: ファイルの編集が反映されない

**原因**: Claude Codeがファイルを編集したが、実際には保存されていない。

**解決方法**:
1. 権限の確認（読み取り専用になっていないか）
2. ファイルが他のアプリケーションで開かれていないか確認
3. Claude Codeの出力をよく読んで、エラーがないか確認
